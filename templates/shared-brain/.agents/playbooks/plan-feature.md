# Plan Feature

Use this workflow before implementing a feature that spans multiple modules or touches contracts / data.

## Read First
1. `.agents/CONTEXT.md`
2. `.agents/MAP.md`
3. `.agents/TASKS.md`
4. `.agents/DECISIONS.md`
5. `.agents/SCHEMA.md` and `.agents/CONTRACTS.md` if relevant

## Produce
1. Scope summary
2. Modules to inspect or modify
3. Data / contract changes
4. Risks and unknowns
5. Ordered implementation plan
6. Suggested `TASKS.md` entries if the user wants the plan operationalized

## Rules
- Inspect source where the map is too coarse.
- Prefer small slices over large speculative plans.
- Separate facts from assumptions.
