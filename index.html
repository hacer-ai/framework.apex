APEX Shared Brain

One memory layer.
Two agents.

APEX is now an installable scaffold for Codex and Claude. The shared brain lives in .agents/, workflows live in .agents/playbooks/, and each tool gets a thin native entry point instead of pretending both runtimes discover the same local skills.

Install the scaffold Read the README

What changed

No more fake cross-tool skill parity Codex is driven by AGENTS.md. Claude is driven by CLAUDE.md and optional command wrappers.
Installable, not just presentational The repo now ships a real template and an installer script.
Shared docs guide exploration The map narrows the search space; it does not replace source validation.
Design rule Shared memory is an accelerator. It is not permission to skip reading the code that is about to change.
Shared layer
.agents/ stores context, map, tasks, progress, decisions, contracts, and schema notes.
Workflow layer
.agents/playbooks/ holds cross-model workflows for session start, wrap-up, planning, and DB work.
Codex layer
AGENTS.md tells Codex how to use the shared brain and which playbook to open for each task shape.
Claude layer
CLAUDE.md points to the same playbooks. Optional .claude/commands/ files wrap them for slash commands.

Install into a real project

./scripts/install-apex.sh /path/to/project --project-name "My Project"
Default behavior Existing files are preserved. The installer skips anything already present unless you pass --force.
Overwrite behavior ./scripts/install-apex.sh . --force
  1. Fill in the placeholders in AGENTS.md, CLAUDE.md, and .agents/.
  2. Commit the scaffold with your project.
  3. Keep shared memory current as code, contracts, and schema evolve.

Scaffold layout

your-project/
├── AGENTS.md
├── CLAUDE.md
├── .agents/
│   ├── CONTEXT.md
│   ├── MAP.md
│   ├── SCHEMA.md
│   ├── TASKS.md
│   ├── PROGRESS.md
│   ├── DECISIONS.md
│   ├── CONTRACTS.md
│   └── playbooks/
│       ├── start-session.md
│       ├── end-session.md
│       ├── db-change.md
│       └── plan-feature.md
└── .claude/
    └── commands/
        ├── apex-start.md
        ├── apex-end.md
        ├── apex-schema.md
        └── apex-plan.md
Important The playbooks are the source of truth for workflows. Claude wrappers are conveniences only.

Compatibility model

Concern Codex Claude
Primary entry point AGENTS.md CLAUDE.md
Shared memory .agents/ .agents/
Workflow source .agents/playbooks/ .agents/playbooks/
Convenience commands Use natural language or agent instructions Optional .claude/commands/ wrappers
Safe assumption Explicit playbook references Explicit playbook references
If your environment supports richer local skill discovery, add it as an optimization. Do not make that optimization the foundation of the framework.
