APEX Shared Brain

One brain.
Humans and agents, aligned.

APEX is an installable scaffold that gives your whole team — humans, Claude, and Codex — a single shared memory layer. Install once, initialize with /apex-init, and every session starts with full context.

Get started Read the README

How it works

1 · Install Run install-apex.sh in your project. Gets the scaffold in place in seconds.
2 · Init Run /apex-init in Claude Code. Claude reads the codebase and fills .agents/ automatically.
3 · Work Use /apex-start to load context, /apex-end to close. Every session stays in sync.
Core rule Shared memory is an accelerator. It is not permission to skip reading the code that is about to change.
Shared memory layer
.agents/ holds context, map, schema, tasks, progress, decisions, and contracts. Every tool and contributor reads the same files.
Workflow layer
.agents/playbooks/ holds model-agnostic workflows. Both Codex and Claude follow the same playbooks.
Codex entry point
AGENTS.md is user-owned. It tells Codex where the shared brain lives and which playbook to open per task shape.
Claude entry point
CLAUDE.md is user-owned. It points to the same playbooks. .claude/commands/ are optional slash-command wrappers.
Active-work signal
Mark tasks with 🔒 ACTIVO: Name · date when working on them. Prevents two agents or contributors from taking the same task.
Session attribution
Every PROGRESS.md entry records who did the work: a human name, Claude, or Codex. Full history, no ambiguity.

Get started in three steps

Step 1 — Install the scaffold
Run from the APEX repo into your project:
./scripts/install-apex.sh /path/to/project --project-name "My Project"
CLAUDE.md and AGENTS.md are never overwritten Those files are yours. The installer creates them only if they don't exist, even with --force.
Refresh framework files ./scripts/install-apex.sh . --force updates playbooks and commands without touching your entry points.
Step 2 — Initialize the shared brain
Open Claude Code in your project and run:
/apex-init
What Claude does automatically Reads your stack, maps modules, detects schema, then asks if you have existing tasks to import before writing anything.
What you do after Review the generated files. Correct anything Claude could not infer. Fill in Known Risks and Decisions manually.
Step 3 — Start your first session
From any session, in Claude Code or Codex:
/apex-start
SESSION LOADED
Project: My App · Mode: ACTIVE
Last completed: T-003 — Auth migration
Active task:    T-004 — Payment webhook
Blocked: none · Risks: none

Playbooks

Playbooks are the source of truth. .claude/commands/ are thin wrappers that open the corresponding playbook. No logic lives in the commands.
Playbook Claude command When to use
init.md /apex-init Once, after install. Claude reads the codebase and fills .agents/. Asks about existing tasks before writing.
start-session.md /apex-start Beginning of every work session. Loads shared brain and surfaces the active task.
end-session.md /apex-end End of every session. Updates PROGRESS, TASKS, MAP, SCHEMA, and suggests a commit message.
plan-feature.md /apex-plan Before implementing anything that spans multiple modules or touches contracts.
db-change.md /apex-schema Any migration, schema edit, query rewrite, or DB-dependent feature. Stops for approval before destructive changes.
onboard.md /apex-onboard When a new human or agent joins. Walks through setup checklist and suggests the first task.

Scaffold layout

your-project/
├── AGENTS.md                    ← Codex entry point (user-owned, never overwritten)
├── CLAUDE.md                    ← Claude entry point (user-owned, never overwritten)
├── .agents/
│   ├── CONTEXT.md               stack, services, env vars, constraints
│   ├── MAP.md                   codebase map + last-verified commit
│   ├── SCHEMA.md                DB view + last-verified commit
│   ├── TASKS.md                 work queue with 🔒 ACTIVO ownership signal
│   ├── PROGRESS.md              session log with human/agent attribution
│   ├── DECISIONS.md             architectural decisions (ADR format)
│   ├── CONTRACTS.md             stable interfaces + change log
│   └── playbooks/
│       ├── init.md
│       ├── start-session.md
│       ├── end-session.md
│       ├── plan-feature.md
│       ├── db-change.md
│       └── onboard.md
└── .claude/
    └── commands/
        ├── apex-init.md
        ├── apex-start.md
        ├── apex-end.md
        ├── apex-plan.md
        ├── apex-schema.md
        └── apex-onboard.md

Compatibility model

Concern Codex Claude
Primary entry point AGENTS.md CLAUDE.md
Shared memory .agents/ .agents/
Workflow source .agents/playbooks/ .agents/playbooks/
Init (fill shared brain) Follow init.md manually or via instructions in AGENTS.md /apex-init slash command
Session commands Natural language: "start session", "end session" /apex-start, /apex-end
Safe assumption Explicit playbook references in AGENTS.md Explicit playbook references in CLAUDE.md
The playbooks are plain markdown committed with your project. If tool support changes, the playbooks remain valid and can be followed manually.
